Message oriented communication

Message-oriented communication based on sockets

Many distributed applications are directly implemented on a simple message-oriented model provided by the transport layer. The implementation takes place directly at this level in form of a communication protocol implementation, without middleware technologies and therefore at a low level of abstraction.

The programming interface for this is the Berkeley Socket API, or in the case of environments such as Java or .Net, a derived environment.

A higher level of abstraction is provided by message queuing, which we will address in the next section.

Note: WebSockets

Please do not associate the term socket directly with WebSockets. The WebSocket protocol is a network protocol designed to establish a bidirectional connection between a web application and a WebSocket server or a web server that also supports WebSockets. The starting point is always an HTTP connection. Websockets will be discussed in detail later.
